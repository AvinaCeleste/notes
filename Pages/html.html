HTML stands for Hyper Text Markup Language. Element names are surrounded by angle brakets to create tags. <tagname>content goes here...</tagname> HTML tags usually come with a start tag and an end tag. The end tag is like the start tag but with a forwards slash before the element name.

HTML documents must start with a document declaration. <!DOCTYPE html> The html tag is a container for your HTML code. The visible part of your code is contained in a body tag. <h1>HEADER</h1>
<p>PARAGRAPH</p>
<a href="#">LINKS</a>
<img src="#" alt="Alternative Text">IMAGES</img>
<button>BUTTON</button>
<ul>UN-ORDERED LIST</ul>
<ol>ORDERED LIST
<li>LIST ITEM 1</li>
<li>LIST ITEM 2</li>
<li>LIST ITEM 3</li>
</ol>
<pre>PREFORMATTED TEXT</pre> HTML elements can be nested within others, as seen in the ordered list example above. Elements that do not contain content are called empty elements. <b>

Attributes provide additional information about an element. Attributes are always defined in the start tag. Using the style attribute, CSS elements can be used in an html document. <body style="color:blue;"></body>

Metadata is undisplayed data about the html document. It typically defines a document title, character set, stylesheets, links and other metadata. <meta charset="UTF-8"> Comments in code can be written using: <!--THIS IS A COMMENT--> CSS can be used to style an HTML page. It can be added to html elements in 3 ways: Inline, uses a style attribute inside an html element; Internal, uses a style element in the head element; External, uses an external CSS file. Id and class attributes are included in html tags to allow individual or groups of elements to be changed by CSS in a way differently than their base tag.

HTML links are called hyperlinks. They are defined using the a tag. Hyperlinks can link to website URLs or they can link to local content. The target attribute specifies the way a link will open: _blank, opens a link in a new tab; _self, opens link in same tab (default); _parent, opens link in parent frame; _top, opens link in full body of document; framename, opens link in named frame. The title attribute creates a tooltip that gives information to users when a link is hovered over. The href attribute can also be used to link to certain element's IDs.

Images can be added to a document using the img tag and the src attribute (used to link to the image source). The alt attribute is used to add an alternative text to an image should there be a loading error or if a user is using a screen reader. HTML allows .gif formats as well. The picture tag can hold multiple image sources that can change based on screen size: <picture>
<source media="(min-width: 650px)" srcset="img_pink_flowers.jpg">
<source media="(min-width: 465px)" srcset="img_white_flower.jpg">
<img src="img_orange_flowers.jpg" alt="Flowers" style="width:auto;">
</picture> An HTML table is defined using the table tag. Inside the table tag, tr (table row) tags can be used. Inside table row tags, td (table data) or th (table head) tags are used, these contain the table content. Using the rowspan or colspan attribute, table content can span across multiple rows or columns. Captions can be added to the table using the caption tag.

There are two types of lists in HTML: unordered lists (ul) and ordered lists (ol). Unordered lists are marked by bullet points while ordered lists are marked by numbers. List items in both types of lists are defined using the li tag nested within the list parent tag. In ordered lists, the type attribute can be used to change the numbering method (1,A,a,I,i). By default, ordered lists will start counting at one. To start counting from a specific number, the start attribute can be used. HTML also supports description lists (dl) The dt tag defines the name and the dd defines the description of each term. Lists can be nested within other lists.

Each HTML element has a default display value depending on its element type. A block-level element always starts on a new line and takes the full width available. Inline elements do not start on a new line and only take up as much space as needed.

Classes allow multiple elements to be styled in the same way outside of their default element tag. HTML elements can have multiple classes (seperated by a space). Different types of tags can also have the same class.

Unlike classes, IDs can only be assigned to one element. IDs can also be used as bookmarks in a page.

Iframes can be used to display a webpage within a webpage. Use a src attribute to link the desired webpage.

JavaScript can be stored with script tags. The script tad can either hold scripting statements or a link to an external script file using the src attribute. The noscript tag allows browsers with disabled scripts to view alternate text.

A filepath describes a location of a file in a website's folder structure. Absolute filepaths point to another website link. Relative file paths point to a file relative to the current page (best practice).

The HTML head tag is a container for metadata: title tag, defines a title in the browser tab and search engine results; style tag, defines style information for a single webpage; link tag, links external stylesheets; meta tag, defines character set, page description, keywords, author, etc.; and base, defines a base url and base target for all relative urls. <meta charset="UTF-8">
<meta name="description" content="web tutorials">
<meta name="author" content="John Doe">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<link rel="stylesheet" href="#"> HTML5 has semantic elements that define different parts of a webpage (header, nace, section, article, aside, footer, details, summary)

Layouts can be built in multiple ways (frameworks, floats, flexboxes or gridview)

Using the viewport meta tag, the browser will automatically reseize the website to the device width. Using a percentage for width, an element will take up that percentage of width relative to screen size. Text size can be set with the vw unit (viewport width).

Computer code elements: kbd, represents input for the user; samp, represents output from a program; code, defines a fragment of computer code; var, defines a variable.

Some characters are reserved in HTML such as > and <. To display these characters, entities must be used. Character sets define the key behing displaying text, numbers, and symbols on a computer. Entity : < Entity name : &lt; Entity Number : &#60;

URL stands for Uniform Resource Locator. A URL is used to address a document on the web. Websites follow syntax rules:
scheme://prefix.domain:port/path/filename
Scheme - type of internet service (http / https)
Prefix - domain prefix (default = www)
Domain - internet domain name
Port - defines port number at the host
Path - path at server (default = root directory)
Filename - name of document / resource

There are several possible schemes: http (HyperText Transfer Protocol), Common and not encrypted; https (Secure HyperText Transfer Protocol), secure and encrypted; ftp (File Transfer Protocol), download and upload files; and file, local file.

XHTML is Extensible HyperText Markup Language. XHTML is almost identical to HTML but stricter. It is supported by all major browsers. It combines the strengths of HTML and XML. In XHTML, doctype, html, head, body and title tags are mandatory. Elements must be properly nested, closed, and in lowercase. For attributes, names must be in lowercase, values must be quoted and minimization is forbidden.

To collect user input form tags are used. The input tag can be displayed in several ways depending on the type attribute.
type=
"text" - one line input field for text.
"radio" - multiple choice allowing one selection.
"submit" - button for submitting for data to form handler.

There are several other ways the type attribute can be used: button, checkbox, color, file, image, number, range, search, time, week, date, datetime-local, email, hidden, month, password, reset, tel, and url.
The required attribute specifies that a field must be filled out before submitting the form. The action attribute defines the action to be performed when the form is submitted. Normally form data is sent to a webpage on the server when the user clicks on the submit button. The method attribute specifies the http method (GET or POST) to be used when submitted: GET, submitted form data will be visible in the page address field (allows users to save searches); POST, submitted form data will not be visible in address bar (used for sensitive or personal material). Each input field must have a name attribute to be submitted. The fieldset tag is used to group related data in a form. The legend tag defines a caption for the fieldset tag. The select tag defines a drop-down list and the option tag (child of select) defines an option that can be selected. <select name="cars>
<option value="volvo">Volvo</option>
<option value="audi">Audi</option>
</select> The size attribute specifies the number of visible values in the select tag. The selected attribute defines a preselected option. The multiple attribute allows users to pick multiple options.

The textarea tag defines a multi-line input field. The rows and cols attribute define the width and height of a text area.

A datalist specifies a list of pre-defined options. Users will se a drop-down list of predefined options as they input data.

The output tag represents the result of a calculation.

When executing scripts in an HTML page, the page becomes unresponsive until the script is finished. A webworker is JavaScript that runs in the background, independently of other scripts, without affecting the performance of the page.
